Research

Development and Planned Implementation of an Improved gem5 Simulator (2017-2018)

  • Worked with Professor Jason Lowe-Power to develop a program to reduce simulation test time. We were developing new ways to identify and separate independent simulation elements so that they may be run in parallel for improved performance.
  • We developed a C wrapper library for ease of access to perf, which is a kernel/syscall interface performance analyzing tool and will be tested with different performance counter APIs (perf, PAPI, Intel PCM) and the PARSEC Benchmark Suite.
  • Ideally, we want to use perf with PARSEC, because perf is more low-level and portable across different hardware. The intention is to eventually use perf with gem5 simulation.

Academic Projects

VR Game for Depression Therapy (Unity, C#, Oculus API, 2018)

  • Worked in a team of 4 people to build a VR game that is based on a mobile app that is currently in development.
  • The game consists of 5 cognitive tasks to train users’ brains to fight depression (the client’s research). In addition, we developed 4 selectable environments, 4 selectable avatars, a rainbow progress tracker, a puzzle incentive to unlock new environments, easily modifiable configuration files, separate user accounts, and a method for data collection.
Facial Recognition (Java, 2017)
  • Wrote a machine learning based image recognition program that learns to distinguish between male and female faces.
  • Input nodes (number of pixels in picture), two output nodes (male/female). Hidden nodes and layers to simplify the neural network and prevent overfitting. Used backpropagation to update the initially random weights in our neural network to minimize the error for each output neuron.
  • Calculated the error for each cycle of the neural network as the AI learned from training images. Used a confidence estimate to determine the accuracy of our AI's predictions.
Connect Four (Java, 2017)
  • Wrote two programs (MiniMax and AlphaBeta algorithm) that compared the 69 winning configurations to the current game board to score player point (points are out of 4 to reflect how close a player is to winning). Created an evaluation function based on the number of player 1 and player 2's odd threats, even threats, and number of game pieces on the board.
  • Tested against a stupid AI, a Monte Carlo AI, and against itself. Always wins against stupid AI and Monte Carlo AI, and only sometimes wins against itself.
Traversing Mount St. Helens (Java, 2017)
  • Wrote an A* search program to find a path from a start point to an end point, while minimizing search space of a terrain of varying heights.
  • Created an admissible heuristic for an exponential cost function that chose a direction to move in depending on cost functions and the surrounding 8 neighbors. The heuristic was to move diagonally until the x- or y- coordinate was equal to that of the goal point, and then move in a straight line to the goal point.
  • Created another admissible heuristic for the division cost function, that obeyed the Triangular Inequality and used the Euclidean distance from the start point to the end point as the shortest path.
